**確認済み事実（出典付き）**:

### 1. Skills のフロントマター拡張（v2.1.0）

**出典**: [Claude Code Docs - Agent Skills](https://code.claude.com/docs/en/skills)、[Hyperdev - Claude Code 2.1.0 ships](https://hyperdev.matsuoka.com/p/claude-code-210-ships)

- ✅ **`model` フィールド対応**: "The frontmatter configures HOW the skill runs (permissions, **model**, metadata)"
- ✅ **`agent` フィールド追加**: エージェント型を指定可能
- ✅ **`context: fork`**: 独立したコンテキストで実行
- ✅ **`user-invocable: false`**: スラッシュコマンドメニューから非表示
- ✅ **YAML形式リスト**: `allowed-tools` を YAML リスト形式で記述可能

### 2. Commands vs Skills の機能差（v2.1.0以降）

| 機能 | Commands | Skills (v2.1.0+) | 判定 |
|------|----------|-----------------|------|
| スラッシュコマンド | ✅ | ✅ | **同じ** |
| model 指定 | ✅ | ✅ | **同じ** |
| agent 指定 | ❌ | ✅ | Skills の方が強力 |
| context: fork | ❌ | ✅ | Skills の方が強力 |
| パラメータ対応 | ✅ `[--option]` | ❓ 不明 | **要確認** |
| allowed-tools | ❌ | ✅ | Skills の方が細かい制御 |
